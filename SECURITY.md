# 보안 정책 (Security Policy)

W-AI Platform (AI-MVP) 프로젝트는 코딩 지식이 부족한 일반인과 소상공인도 안전하게 다중 LLM 마케팅 파이프라인을 사용할 수 있도록 최고 수준의 보안을 지향합니다.

## 지원되는 버전

보안 업데이트가 지원되는 프로젝트 버전은 다음과 같습니다.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

보안 취약점을 발견하셨다면, 공개적인 GitHub Issue로 등록하지 마시고 즉시 아래 이메일로 메인테이너에게 연락해 주시기 바랍니다.

* **Email**: fosum@kakao.com
* 메일 확인 후 48시간 이내에 접수 및 조치 계획을 회신해 드립니다.

## 주요 보안 대응 가이드라인

본 프로젝트는 여러 외부 LLM(ChatGPT, Claude 등)과 블로그 API 키를 다루므로 다음 보안 수칙을 엄격히 적용합니다.

### 1. API 키 및 민감한 데이터 (Secrets) 관리
* **하드코딩 금지**: 코드 내에 어떠한 API 키도 직접 하드코딩해서는 안 됩니다.
* **환경 변수 사용**: 모든 인증 정보는 `.env` 파일을 통해서만 관리되어야 하며, `.env` 파일은 절대 Git 레포지토리에 커밋되지 않도록 `.gitignore`에 등록되어 있습니다.
* 오픈소스 사용자는 반드시 `example.env`를 복사하여 자신의 환경에 맞게 로컬에서만 키를 주입해야 합니다.

### 2. 프롬프트 인젝션 (Prompt Injection) 방어
* 악의적인 사용자가 AI 에이전트의 시스템 프롬프트를 우회하거나 조작하는 것을 방지하기 위한 입력값 검증(Sanitization) 로직을 지속적으로 강화하고 있습니다.
* **Codex Security**: OpenAI Codex Security 프로그램을 통해 코드베이스 전반의 보안 취약점을 사전 검토하고 반영할 계획입니다.
